## 2. Введение и архитектура

Бот \"Секретарь Клава\" --- это интеллектуальный голосовой помощник,
работающий через Telegram. Он принимает голосовые сообщения, распознает
их, анализирует содержание с помощью LLM и возвращает структурированный
ответ.

**Архитектура:**

- **Telegram Bot:** Фронтенд для взаимодействия с пользователем.

- **faster-whisper:** Локальная модель для преобразования речи в текст
  (работает на CPU).

- **DeepSeek API:** \"Мозг\" бота для классификации сообщений (задача,
  встреча, заметка и т.д.) и извлечения сущностей.

- **PostgreSQL:** База данных для хранения пользователей и задач (хотя в
  финальной версии мы ее отключили для простоты).

- **Docker:** Контейнеризация всего стека для простоты развертывания.

## 3. Подготовка сервера

**Требования к серверу (минимальные):**

- **ОС:** Ubuntu 22.04 / 24.04

## 11. Решение распространенных проблем

### 11.1. Ошибка \"Message is not modified\"

- **Причина:** Попытка отредактировать сообщение тем же текстом.

- **Решение:** Использовать функцию safe_edit_text, которая сравнивает
  текущий текст с новым перед отправкой.

### 11.2. Ошибка \"column users.last_name does not exist\"

- **Причина:** Модель User в базе данных отличается от той, которую
  ожидает SQLAlchemy.

- **Решение (временное):** Полностью отключить логику сохранения в базу
  данных в обработчике voice_handler.py (как сделано в финальной
  версии).

### 11.3. Ошибка \"402 Payment Required\" от DeepSeek

- **Причина:** Недостаточно средств на аккаунте DeepSeek.

- **Решение:** Пополнить баланс
  на [platform.deepseek.com](https://platform.deepseek.com).

### 11.4. Медленная работа распознавания

- **Причина:** Модель large-v3 слишком тяжела для CPU.

- **Решение:** Использовать модель small или medium, как в финальной
  версии.

### 11.5. Ключ DeepSeek не виден в контейнере

- **Причина:** Переменные окружения не передаются из .env в контейнер.

- **Решение:** Убедиться, что в
  секции environment файла docker-compose.yml для сервиса botдобавлены
  строки:

> yaml
>
> environment:
>
> \- DEEPSEEK_API_KEY=\${DEEPSEEK_API_KEY}
>
> \- DEEPSEEK_BASE_URL=\${DEEPSEEK_BASE_URL}
