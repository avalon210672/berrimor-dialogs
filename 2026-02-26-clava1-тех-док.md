## 2. Введение и архитектура

Бот \"Секретарь Клава\" --- это интеллектуальный голосовой помощник,
работающий через Telegram. Он принимает голосовые сообщения, распознает
их, анализирует содержание с помощью LLM, сохраняет задачи в базу данных
и возвращает структурированный ответ.

**Архитектура:**

- **Telegram Bot:** Фронтенд для взаимодействия с пользователем.

- **faster-whisper:** Локальная модель для преобразования речи в текст
  (работает на CPU, модель **small**).

- **DeepSeek API:** \"Мозг\" бота для классификации сообщений (задача,
  встреча, заметка и т.д.) и извлечения сущностей.

- **PostgreSQL:** База данных для хранения пользователей и задач.

- **Docker:** Контейнеризация всего стека для простоты развертывания.

- **Единый конфигурационный файл:** Централизованное управление
  действиями через actions_config.py.

## 7. Модули и их назначение

### 7.1. Сервис DeepSeek (deepseek_service.py)

Модуль для взаимодействия с DeepSeek API. Поддерживает два режима:

- analyze_message - стандартный анализ

- analyze_with_prompt - анализ с кастомным промптом

### 7.2. Промпты (secretary_prompts.py)

Содержит основной промпт для DeepSeek с инструкциями по классификации
сообщений и извлечению данных.

### 7.3. Конфигурация действий (actions_config.py) - **ЦЕНТРАЛЬНЫЙ МОДУЛЬ**

Единое место для управления всеми действиями бота:

python

ACTION_TYPES = {

\"add\": \[\"add\", \"create\", \"new\"\], *\# создание задач*

\"status\": \[\"status\", \"list\", \"show\", \"what\"\], *\# запросы
статуса*

\"schedule\": \[\"schedule\", \"calendar\"\], *\# запросы расписания*

\"ask\": \[\"ask\", \"question\", \"who\", \"what\"\], *\# вопросы о
боте*

\"complete\": \[\"complete\", \"done\"\], *\# отметка о выполнении*

\"delete\": \[\"delete\", \"remove\"\], *\# удаление задач*

}

*\# Декоратор для регистрации обработчиков*

\@register_handler(\"add\")

async def handle_add_action(\...): \...

\@register_handler(\"status\")

async def handle_status_action(\...): \...

\@register_handler(\"ask\")

async def handle_ask_action(\...): \...

*\# Фабрика выполнения*

async def execute_action(\...): \...

### 7.4. Обработчик голоса (voice_handler.py)

Основной файл, который:

1.  Скачивает голосовое сообщение

2.  Распознает речь через faster-whisper

3.  Анализирует через DeepSeek с персонализированным промптом

4.  Определяет действие и передает управление в actions_config

### 7.5. Работа с базой данных (task_crud.py)

Прямые SQL-запросы к таблице tasks (без использования ORM моделей).

## 9. Интеграция локального распознавания речи (faster-whisper)

Используется модель **small** для оптимального соотношения скорости и
качества на CPU.

**Ключевые параметры:**

- Модель: small

- Устройство: cpu

- Тип вычислений: int8 (квантизация для скорости)

## 14. Решение распространенных проблем

### 14.1. Ошибка \"Message is not modified\"

**Решение:** Использовать функцию safe_edit_text, которая сравнивает
текущий текст с новым перед отправкой.

### 14.2. Ошибка \"ForeignKeyViolation\" (user_id)

**Причина:** В таблицу tasks передается Telegram ID вместо внутреннего
ID из таблицы users.\
**Решение:** Использовать правильное преобразование:

python

user_id_in_db = 1 if user.id == 884710380 else 2 if user.id == 493002928
else None

result = task_service.create_from_analysis(user_id_in_db, task_data)

### 14.3. Ошибка \"ModuleNotFoundError: No module named \'transkribator_modules.config\'\"

**Причина:** Конфликт между папкой config и файлом config.py.\
**Решение:** Переименовать папку в actions_config и
создать \_\_init\_\_.py.

### 14.4. Медленная работа распознавания

**Причина:** Модель large-v3 слишком тяжела для CPU.\
**Решение:** Использовать модель small или medium.

### 14.5. DeepSeek возвращает action=\'ask\' вместо \'add\'

**Причина:** Неправильный промпт.\
**Решение:** Использовать обновленный промпт в secretary_prompts.py с
четкими инструкциями.
